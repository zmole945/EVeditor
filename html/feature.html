How to use features

  • Go to matching bracket

    Select "Navigate"->"Go to"->"Matching Bracket" menu.
    Then cursor position goes to opposite bracket of "[]", "{}", "()" and "begin/end"

  • Contents outline view

    Select "Window"->"Show View"->"Outline".
    The cursor position goes to the outline element with double clicking.

  • Open declaration

    Go to module declaration of selected word by searching a project hierarchy.
    It is valid when you defined a same module name with file name, e.g. module "adder" is defined in file "adder.v".
    Select "Navigate"->"Open Declaration" menu, "Open Declaration" in popup menu of editor page.

  • Content assist

    Enter a few character and enter Ctrl+space or "Edit"->"Content Assist" menu.
    Select word from a proposal list.

    The proposal list may include code templates, signals and module instances, which depend on Verilog context.

  • Module Hierarchy

    Select "Window"->"Show View"->"Other..." menu, then select "Module Hierarchy" in "Eclipse Verilog Editor" category.

    You can open editor and go to declaration by double clicking on Module Hierarchy view.

    You can open popup menu on Module Hierarchy view by right button clicking. The menu show "copy text" and "copy hierarchy". The "copy text" copies module list to clipboard simply. The "copy hierarchy" copies hierarchical list.

  • Problems view

    You have to install Icarus Verilog, CVer or FreeHDL at first. Then you can select "Edit"->"Compile". You will see error and warning messages in Problems view if exist.
    Warning: it is just simple method. If your design has many modules and libraries, you should use "Verilog/VHDL simulator builder".

  • Preference page

    You can select "Window"->"Preferences"->"Verilog/VHDL Editor".

  • Code Formatter

    You can select "Edit"->"Format".

  • Console view

    Console view automatically opens when you compile or parser error occurs.

  • Verilog/VHDL perspective

    Select "Window"->"Open Perspective"->"Other...", then select "Verilog/VHDL".
    Then you will see "Verilog/VHDL" icon on top left corner on Eclipse Window.

  • Verilog/VHDL new project wizard

    In the Verilog/VHDL perspective, select "File"->"New"->"Project" or "New"->"Project" on Navigator. Then select "Verilog/VHDL project" and push "Next". Enter Project name and push "Finish".

  • Verilog/VHDL new file wizard

    In the Verilog/VHDL perspective, select "File"->"New"->"Verilog file" or "New"->"Verilog file" on Navigator. Then enter module name and push "Finish". You can also select "VHDL file".

  • Verilog/VHDL simulator builder

    Select your Verilog/VHDL project on Navigator. Then open context menu and select "Properties". Select "Verilog/VHDL Simulator" category. Check "Enable Verilog/VHDL Builder" and fill "Working Folder", "Simulate command" and "Arguments". Finally you have to select "Error parser" which are Icarus Verilog, CVer or FreeHDL now. Push "OK"

    Then you can "Build project". Problems view reflects the simulator error and warning messages.

    Warning:
    Eclipse builds automatically by default. So you save your Verilog or VHDL file, your simulator run immediately. To avoid this, you can disable "build automatically". To disable, you have to select "Project"->"Build automatically". When you want to simulate in build non-automatically, you can select "Project"->"Build->Project" or "Project"->"Build All".

  • Custom error parser

    "Window"->"Preferences"->"Verilog/VHDL Editor"->"Error Parser". Push "New" button, then you can add your favorite complier.
    You have to fill text box named "Error Pattern". It is matching string for regular expression of java.util.regex package. The first capturing group means the file name, second is the line number and third is error message.

    If you add your error parser, you can select it in the project properties page.

    The document of java.uitl.regex package is http://java.sun.com/j2se/1.4.2/docs/api/java/util/regex/package-summary.html. The Pattern class link shows how to write the regular expression.
